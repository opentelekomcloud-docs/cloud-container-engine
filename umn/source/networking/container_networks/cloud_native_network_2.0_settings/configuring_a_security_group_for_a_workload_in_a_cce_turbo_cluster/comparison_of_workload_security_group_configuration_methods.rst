Comparison of Workload Security Group Configuration Methods
===========================================================

In CCE Turbo clusters, pods can be directly bound to security groups using VPC network interfaces or supplementary network interfaces. CCE Turbo provides multi-dimensional security group binding methods to meet your service needs.

.. caution::

   If multiple security group configuration methods are used, the method with the highest priority will be applied. In the table below, smaller values indicate higher priorities.

.. table:: **Table 1** Comparison between methods for configuring workload security groups

   +-----------------+-----------------------------------------------------------------------------------------------------------------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------------+

   +=================+=============================================================================================================================+====================================================================================================================================================================+===========================================================================================================================+
   | 1               | :ref:`Binding a Security Group to a Pod Using an Annotation <cce_10_0897>`                                                  | -  This method is suitable for debugging services.                                                                                                                 | -  Newly added security groups apply only to new pods. To apply the configuration to existing pods, they must be rebuilt. |
   |                 |                                                                                                                             | -  Security groups added using this method can be used by other configuration methods.                                                                             | -  Pre-bound container network interfaces cannot be associated with a target security group.                              |
   +-----------------+-----------------------------------------------------------------------------------------------------------------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------------+
   | 2               | :ref:`Binding a Security Group to a Workload Using a Security Group Policy <cce_10_0288>`                                   | -  This method applies to the entire namespace and requires workloads within the namespace.                                                                        | Pre-bound container network interfaces cannot be associated with a target security group.                                 |
   |                 |                                                                                                                             | -  Security group settings can be updated for pods in real time. This eliminates the need to rebuild existing pods.                                                |                                                                                                                           |
   +-----------------+-----------------------------------------------------------------------------------------------------------------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------------+
   | 3               | :ref:`Using Node Pool Settings to Bind the Default Security Group to Pods in the Node Pool <cce_10_1079>`                   | -  Services are classified by node pool. You may need to configure associated scheduling policies based on your services.                                          | -  Newly added security groups apply only to new pods. To apply the configuration to existing pods, they must be rebuilt. |
   |                 |                                                                                                                             | -  Pre-bound container network interfaces can be associated with a target security group.                                                                          | -  Only newly bound container network interfaces on the node can be associated with a target security group.              |

To apply the configuration to existing pods, they must be rebuilt. |
